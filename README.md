# Мультиагентная аналитическая система с инструментами и RAG-модулем

## Как устроена оценка

Проект - это **конструктор**: ниже — список компонентов с баллами. Берите любую комбинацию под свою задачу и свой домен, набирайте нужную сумму баллов. Никакой компонент сам по себе не обязателен — кроме блока «Обязательный минимум».

---

## Обязательный минимум (без баллов за факт наличия, но без этого проект не принимается)

- **API (REST/gRPC) \+ простой веб\-интерфейс**  
- **Деплой**: облако или локально, Docker/Docker Compose, инструкции запуска  
- **Мониторинг/логирование/трейсинг** (например, Prometheus+Grafana)  
- **Репозиторий на GitHub**: код, Docker/Compose, инструкции запуска  
- **Дизайн-документ** (шаблон: [https://colab.research.google.com/drive/1KBG74ca-fLKqORUZxfLiIQiNP1FvH0ZN?usp=sharing](https://colab.research.google.com/drive/1KBG74ca-fLKqORUZxfLiIQiNP1FvH0ZN?usp=sharing)), описывающий: какую задачу решает система, какие компоненты из корзины ниже выбраны и почему  
- **Ноутбуки с экспериментами**, подтверждающие выбранные из корзины пункты (чанкинг/поиск/ранжирование/обучение/валидация — те, что реально делали)  
- Ссылка на демо-видео 3–5 мин и/или публичный endpoint (если возможно)

Это инфраструктурный костяк — без него непонятно, что и как оценивать. Сам факт наличия не приносит баллов сверху, но без него проект не рассматривается.

## Корзина компонентов

Набираете баллы из любых разделов ниже в любой комбинации.

### Архитектура и оркестрация 

| Компонент | Баллы |
| :---- | :---- |
| Система из ≥3 координируемых LLM-агентов с разделением ролей (Planner/Researcher/Analyst/Reviewer/Reporter или свой набор) | 3 |
| Коммуникация с валидацией по схемам (SGR/Structured Output) | 1 |
| Шина сообщений или адресные ask\_agent/reply с логами входящих/исходящих сообщений | 2 |

*Если вместо мультиагентной системы делаете одного агента с инструментами и RAG — этот раздел просто не берёте, баллы добираете из других.*

### Дополнительные агентские возможности 

| Компонент | Баллы |
| :---- | :---- |

| Устойчивость к ошибкам инструментов (retry/fallback при сбое tool-call) | 3 |

| Компонент | Баллы |

### Модуль знаний и поиск (RAG)

| Компонент | Баллы |
| :---- | :---- |

## Итог: как считается оценка

Шкала — десятибалльная. Итоговая оценка \= **MIN(10, сумма набранных баллов по корзине)**.

Обязательный минимум (инфраструктура, мониторинг, репозиторий, дизайн-документ) баллов не даёт и в сумму не входит — без него проект просто не идёт на оценку.
